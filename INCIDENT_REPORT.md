# Отчёт об устранении инцидентов

## Проект

MediTrack Desktop

## Выявленные проблемы

В ходе анализа проекта были выявлены следующие проблемы:

* параметры подключения к базе данных не должны храниться непосредственно в исходном коде;
* локальные учетные данные необходимо хранить во внешнем конфигурационном файле `.env`;
* для диагностики ошибок подключения к базе данных требуется система логирования;
* документация проекта должна содержать инструкцию по настройке окружения и запуску приложения.

## Выполненные изменения

В рамках работы были реализованы следующие улучшения:

* настроено хранение параметров подключения к базе данных через файл `.env`;
* настроена система логирования через `LoggerConfig.cs`;
* добавлено логирование операций работы с базой данных в `DatabaseService.cs`;
* обновлена документация проекта (`README.md`);
* описана настройка PostgreSQL и необходимые переменные окружения.

## Переменные окружения

```env
DB_HOST=localhost
DB_PORT=5433
DB_NAME=meditrack_db
DB_USER=postgres
DB_PASSWORD=YOUR_PASSWORD
```

## Проведённое тестирование

Для проверки работоспособности были выполнены следующие действия:

1. Создана база данных PostgreSQL.
2. Выполнен SQL-скрипт `init_db.sql`.
3. Создан файл `.env` с параметрами подключения.
4. Выполнена сборка проекта.
5. Выполнен запуск приложения.
6. Проверено подключение к базе данных.
7. Проверена работа системы логирования.

## Результат

Проект подготовлен для локального развертывания и дальнейшего сопровождения. Подключение к базе данных вынесено в конфигурационный файл, система логирования настроена и готова к использованию, документация обновлена.
