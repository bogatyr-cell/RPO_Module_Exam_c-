Проблема №1. Отсутствие поддержки переменных окружения
Симптом

При использовании методов библиотеки DotNetEnv проект не собирался и выдавал ошибку компиляции.

Причина

В проекте отсутствовал пакет DotNetEnv.

Решение

Выполнена установка необходимой библиотеки:

dotnet add package DotNetEnv

После установки проект получил возможность работать с переменными окружения через файл .env.



Вставить скриншот ошибки компиляции до установки пакета.

Проблема №2. Не считываются параметры подключения из .env
Симптом

Приложение не получало значения переменных окружения и не могло корректно сформировать строку подключения к PostgreSQL.

Причина

Отсутствовал вызов загрузки файла окружения.

Решение

В конструктор класса DatabaseService была добавлена команда:

Env.Load();

После внесения изменений приложение начало корректно получать параметры подключения из файла .env.



Вставить скриншот конструктора DatabaseService после внесения изменений.

Проблема №3. Ошибки не сохраняются в журнал
Симптом

При возникновении исключения отображалось только окно сообщения пользователю, однако информация об ошибке отсутствовала в логах приложения.

Причина

В блоке обработки исключений отсутствовала запись ошибки через Serilog.

Решение

В блок catch добавлена запись ошибки:

Log.Error(ex, "Ошибка при загрузке данных из базы PostgreSQL.");

Теперь все исключения сохраняются в лог-файл и могут использоваться для диагностики.



Вставить скриншот блока catch после внесения изменений.

Проблема №4. Недостаточная диагностическая информация
Симптом

В журнале фиксировался только текст ошибки без подробной информации об исключении.

Причина

Объект Exception не передавался в систему логирования.

Решение

В вызов Log.Error был добавлен объект исключения ex.

Это позволило сохранять:

тип ошибки;
текст исключения;
стек вызовов;
дополнительную служебную информацию.


Вставить скриншот лог-файла с отображением полной информации об ошибке.

Проблема №5. Файл .env не обнаруживается приложением
Симптом

Приложение не могло получить значения переменных окружения.

Причина

Файл .env находился вне каталога запуска приложения.

Решение

Файл .env был перенесён в рабочую директорию приложения (bin/Debug), после чего параметры окружения стали успешно считываться.


Вставить скриншот расположения файла .env.
